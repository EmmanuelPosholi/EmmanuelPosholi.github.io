## Reflection Coding In Markdown

I was quite concerned when I needed to write my CV in GitHub in Markdown, as I had never written in Markdown. The situation was that I had to write my CV in markdown and publish it as it was part of my e-portfolio on GitHub. My assignment was to effectively code my complete CV with Markdown syntax, headings, lists, links and formatting without errors.

Initially, the idea of writing in Markdown somehow worried me as I was uncertain how dissimilar it would be to regular writing, and I feared that I would have issues with the syntax. But, after I began, I found out that Markdown coding was not as difficult as I had supposed. The syntax was user-friendly and easy to learn, and I learnt it in a very short period of time. It was easy to comprehend, and hence I carried out all the things without any difficulty. Eventually, I was able to code my CV in Markdown without much problem, and it appeared as I intended it to be.

## Reflection On Mock Interview

The most difficult part of my e-portfolio was including the mock interview. The situation was that I needed to record and include a mock interview as part of my e‑portfolio, which I was deploying on GitHub Pages alongside my CV. My task was to complete the mock interview portion and integrate it into my e‑portfolio so that everything looked professional. At first, I found this part extremely challenging because I was overthinking it. I kept worrying about what to say and whether I sounded good enough or professional enough. 

Because I was overthinking so much, I ended up doing many takes before I felt even close to being satisfied. Every time I recorded, I would listen back and find something wrong, so I would delete it and try again. This happened over and over again, and it started to feel frustrating. However, I did not give up. After plenty of tries, I finally stopped overthinking and just let myself speak naturally. Once I did that, everything came together, and I successfully executed the mock interview. 

In the end, I was satisfied with the final version, and I was able to include it in my e-portfolio. Even though the mock interview was the hardest part of the whole project, pushing through all those takes taught me that overthinking only makes things harder, and persistence pays off.

## Reflection on Github Pages

I was supposed to deploy my e-portfolio on GitHub Pages, and my e-portfolio consisted of a CV and a mock interview. Initially, I had no idea how to use GitHub Pages since it was the first time I had ever done so. I was confused in the entire process, and I was not even aware of where to begin. 

However, then I did watch a YouTube guide on how to use GitHub Pages and after viewing that YouTube tutorial, my head was opened. The YouTube tutorial was a step-by-step guide of what I had to do and once I went through it, I was able to see how I could use GitHub Pages to deploy my e-portfolio. In the end, I successfully used GitHub Pages to deploy my e‑portfolio, which included my CV and mock interview.

Although initially it was hard to follow as it was my first experience with GitHub Pages, watching the tutorial on YouTube helped open my eyes and realize how to do it.
